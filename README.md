# Kotlin-MVVM
안드로이드 디자인패턴: MVVM

### View : UI 요소를 표시하며 사용자가 발생한 이벤트를 받는 역할  
### ViewModel : UI 요소에 들어갈 데이터를 관리합니다. 그리고 Model과 View 사이의 다리 역할  
### Model : 데이터 전반적인 것을 처리하는 역할
- Repository
- Retrofit / Room
